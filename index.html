How to use — IT Helpdesk Tool
A guide to every tab and feature · v3.14
Password & Access

The site is password protected. Enter sceptre to unlock. Once unlocked it stays open for the rest of your browser session — you won't need to re-enter it unless you close and reopen the browser.

The tool is hosted at fraaz-kagzi.github.io/IT_Helpdesk_Tool and works on any NHS laptop in any browser. No install required. All data is stored locally in your browser — nothing is sent externally.

Personalisation Bar

Sits at the top of every page. Enter your name once and every script you copy will automatically sign off as Your Name — IT Helpdesk. Your name is remembered between sessions.

Optionally enter the customer's name to personalise the greeting in scripts (Hi [Name],). Clear this between tickets so one customer's name doesn't carry over to the next.

Tab 1
Script Finder

A library of 129 pre-written helpdesk scripts covering the most common issues — printers, access requests, MFA, Teams, Egress, RiO, EMIS, known inactive, leavers, and more.

  • Search bar — type a few words (e.g. "canon", "drive access", "teams") to instantly filter. Results are ranked by relevance.
  • Category tags — click a tag (access, printer, mobile, email etc.) to filter by topic. Combine with a search for more precision.
  • Copy — copies the full script with your name and the customer name already filled in. Paste straight into Sostenuto or an email.
  • Edit — modify any script inline. Changes save locally.
  • Delete — removes a custom script permanently.
  • Add new script — create your own at the bottom of the tab. Give it a name, comma-separated keywords (used for search), and a body. Use {{agent}} where you want your name and {{customer}} for the customer greeting.
  • Import scripts — upload a .txt file to bulk-import scripts. Supports the [label] format used in the original IT_Prompts.txt file.
  • Team Files — download the latest RA spreadsheet and scripts file from GitHub with one click.
Tab 2
Known Inactive

Quickly generates the standard output message for processing a known inactive account.

  • Type the account name as Firstname.Lastname — the tool appends @lscft.nhs.uk automatically.
  • Select the OU (North, East, West, Central) the account should be moved to.
  • The output is generated instantly: Firstname.Lastname@lscft.nhs.uk details changed and moved to [OU] OU
  • Click Copy to copy it. Entries are logged in the Recent Entries history below — click any entry to re-copy it.
Tab 3
RA Agents

A searchable directory of all LCFT Registration Authority (RA) agents — the local staff who can unblock smartcards, reset passcodes, and assist with certificate expiry.

  • Search — type a site name, town, postcode, or agent name. All matching sites and agents appear instantly.
  • Each agent card shows their phone number, email (click to copy), and any notes such as availability hours or RA support at weekends.
  • Agents with a Repairs Certificates badge can fix expired smartcard certificates.
  • Upload spreadsheet — if the RA list has been updated, upload the new Local_RA_Admins.xlsx (or a CSV version) to refresh the data. It saves to your browser and loads automatically next time.
  • The built-in data covers 174 agents across all LCFT sites.
Tab 4
Passwords

A local bank of temporary passwords to hand out to users who need a reset. Passwords can be reused freely.

  • Copy — click the Copy button on any password card. It flashes green for confirmation then resets, ready to copy again.
  • Favourite — star a password to pin it to the top of the list.
  • Add manually — type a password at the bottom and press Enter or click Add.
  • Upload .txt file — load a plain text file with one password per line to replace or add to the list.
  • Clear all — wipes the entire list.
  • The default list includes the 8 standard temp passwords. The list persists in your browser between sessions.
Tab 5
Site Info

Look up any LSCFT site to find the field technician area, field technician, and remote technician assigned to it. Covers all 155 sites in the LCFT Site Directory.

  • Search — type part of a site name, town, or tech name. Results update as you type.
  • Useful when a user calls and you need to know who to book a field tech appointment with, or which remote tech covers their area.
Tab 6
MFA Resync

Provides ready-to-run PowerShell commands for re-syncing a user's MFA registration in Active Directory.

  • Two command blocks are available — the standard script and the v3.14 version. Click Copy Command on the relevant one and paste it into PowerShell opened as your standard AD account.
  • You will be prompted to enter the user's username and if they are un-synced, a Sostenuto job number.
  • Expand the How to use section on that tab for a full step-by-step walkthrough.
Tab 7
Manual Log

An emergency job-logging form for use when Sostenuto is unavailable. Logs jobs locally so nothing is lost.

  • Fill in the caller details, site, division, asset number, and job description. Required fields are marked with a red * — the form won't submit without them.
  • The site field has autocomplete — start typing and matching LCFT sites will appear as suggestions.
  • Each job gets an auto-generated reference number in the format MAN-YYYYMMDD-XX.
  • Logged jobs appear on the right. Click × on any job to remove it individually.
  • Export as XLSX — downloads a proper Excel file with all jobs and a bold header row.
  • Export as CSV — downloads a plain comma-separated file compatible with any spreadsheet app.
  • Clear All Jobs — removes all logged jobs after confirmation.
  • Jobs persist in your browser between sessions until you export or clear them.
Tips & Notes
  • All data (scripts, passwords, RA agents, logged jobs) is stored in your browser's local storage. It persists between sessions but is specific to your device and browser — it won't sync to a colleague's machine.
  • If a colleague opens the site and sees fewer scripts than expected, they may have an older version cached. The scripts will reset to the latest defaults automatically when a new version is published.
  • The site works fully offline once loaded — no internet is required after the initial page load.
  • Use the Team Files bar at the top to download the latest shared files (RA spreadsheet, scripts file) directly from GitHub.
  • The boss banner at the top is clickable — it opens the full image.

MFA Resync

Copy the command below and paste it straight into a PowerShell window opened as your standard AD admin account.

PowerShell Command
cd \\lct-sdadmin01\Tools\MFA_Script; .\MFA_Resyncer.ps1 -RunForReal Copied!
PowerShell Command — v3.14
cd \\lct-sdadmin01\Tools\MFA_Script; .\MFA_Resyncer-test.ps1 -RunForReal Copied!
▼ How to use the MFA Resync script

Script location:
\\Nct-sdadmin01\Tools\MFA_Script

Steps:

  1. Open a PowerShell window as your standard user (the account you use for day-to-day Active Directory work).
  2. Navigate to the script directory:
    cd \\Ict-sdadmin01\Tools\MFA_Script
  3. Run the script:
    .\MFA_Resyncer.ps1 -RunForReal

Note: Without -RunForReal the script runs in simulation mode and makes no changes — useful for previewing what it would do.

At the prompt:

  • Enter the username of the user to re-sync and press Enter.
  • If the user is already synced, you will see a confirmation message.
  • If the user is un-synced, you will be prompted for a SOS job number — this is added to the notes field in AD. Enter the job reference and press Enter.
